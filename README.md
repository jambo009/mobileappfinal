🧊 나의 냉장고 (My Fridge)
"버려지는 식재료 제로! 자취생을 위한 스마트 유통기한 관리 솔루션"

안드로이드 Jetpack Compose를 활용하여 자취생들이 냉장고 속 식재료를 직관적으로 관리하고, 불필요한 식재료 낭비를 줄여 식비를 절감할 수 있도록 돕는 실무형 애플리케이션입니다.

🚀 개발 배경 및 의도 (Motivation)
많은 자취생이 냉장고 깊숙이 있는 식재료의 유통기한을 잊어버려 음식을 버리곤 합니다. 기존 메모장 앱은 매번 남은 일수를 계산해야 하는 번거로움이 있었습니다. 저는 이 **'계산의 번거로움'**을 자동화하고, **'시각적 경고'**를 통해 즉각적인 행동(소비)을 유도하는 앱을 직접 기획하고 개발했습니다.

✨ 핵심 기능 (Key Features)
1. 세션 기반 관리 (Session-Based Workflow)
관리 모드 활성화: 앱 시작 시 '관리 시작하기' 버튼을 배치하여 사용자가 명확한 목적(냉장고 점검)을 가지고 세션을 시작하도록 설계했습니다.

직관적 종료: 상단 툴바의 종료 버튼을 통해 점검을 완료하는 일련의 워크플로우를 구축했습니다.

2. 유통기한 신호등 시스템 (Color-Coded Status)
자동 D-Day 계산: 등록된 날짜와 현재 시간을 비교하여 잔여 일수를 실시간으로 연산합니다.

시각적 우선순위: 남은 기간에 따라 리스트 아이템의 색상이 자동으로 변경되어, 무엇을 먼저 먹어야 할지 한눈에 파악할 수 있습니다.

🔴 빨강 (만료/긴급): 유통기한이 지났거나 3일 이내인 경우

🟡 노랑 (주의): 유통기한이 7일 이내인 경우

🟢 초록 (안전): 유통기한이 넉넉한 경우

3. 실시간 요약 대시보드
긴급 품목 브리핑: 메인 화면 상단에 '임박 식품 개수'를 자동으로 집계하여 보여줌으로써 사용자의 주의를 환기합니다.

🛠 사용 기술 및 문제 해결 (Tech Stack & Problem Solving)
1. 기술 스택
Language: Kotlin

UI Framework: Jetpack Compose (Material 3)

State Management: mutableStateOf, remember를 통한 반응형 UI 구현

2. 기술적 도전과 해결 (Troubleshooting)
상태 동기화 문제: 리스트 내의 아이템이 삭제되거나 추가될 때 화면이 즉시 갱신되지 않는 문제를 해결하기 위해, mutableStateOf에 새로운 리스트 객체를 할당하는 방식으로 Compose의 재구성(Recomposition)을 유도했습니다.

문자열 템플릿 최적화: 코틀린 문자열 내에서 변수와 한글 조사가 붙어있을 때 발생하는 인식 오류를 ${variable}개와 같은 중괄호 표기법을 사용하여 해결했습니다.

데이터 관리: FoodItem 데이터 클래스 내에 daysLeft 연산 로직을 직접 포함시켜 UI 코드와 비즈니스 로직을 분리(Separation of Concerns)함으로써 코드 가독성을 높였습니다.

🏗 프로젝트 구조 (Structure)
코드의 유지보수성을 위해 기능별로 파일을 분리하여 관리했습니다.

MainActivity.kt: 앱의 전체 흐름 및 세션(시작/종료) 상태 제어

FoodItem.kt: 식재료 데이터 모델 및 유통기한 판별 알고리즘

Components.kt: 재사용 가능한 UI 컴포넌트 (카드 레이아웃, 입력 다이얼로그)
